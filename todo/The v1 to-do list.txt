The v1 To-Do List (Weeks 1–3: Prove the Pipeline)

Goal: Build one hardcoded pipeline from RSS to a deployed MCP search server. No visual canvas yet.

1. Data Ingestion (RSS + Audio + Web Scraping)
- Accept RSS URL input from the user
- Accept Website url as well
- Scrape website url comepletely (including all the subpages as well if available in the website provided)
- Fetch the RSS feed and parse out episode titles, descriptions, and MP3 links
- Download the MP3 audio files
- Integrate with Whisper API (OpenAI) to transcribe audio files to plain text
- Handle errors, malformed feeds, and timeouts gracefully


Indepth process

RSS ingestion

 Given any RSS URL, fetch the feed
 Extract title, date, description, and link for every item
 Handle malformed or slow feeds gracefully with a timeout
 Return a clean list of episodes/posts with their full text content

Web scraping

 Given any webpage URL, fetch the HTML
 Strip all HTML tags, navigation, headers, footers — extract only the readable body text
 If the URL has links to subpages (like a documentation site), crawl one level deep and collect all linked pages
 Detect if a URL is an RSS feed or a webpage and route accordingly





2. Processing Pipeline (Chunk & Embed)
- Chunker: Split transcript text into chunks of ~400 words with a 50-word overlap
- Embedder: Convert each text chunk into a vector using an embedding model (e.g., OpenAI text-embedding-3-small)
- Vector DB: Set up Cloudflare Vectorize (or Supabase pgvector) to store chunks, vectors, and metadata (episode title, date, podcast name)

3. MCP Server Code Generation
- Create a template for a TypeScript MCP server
- Code Generator: Take the server config (DB credentials, etc.) and produce the final TypeScript server code
- The generated server must expose one tool: `search_content(query: string)`
- Implement semantic search within the generated server to query the Vector DB and return formatted results

4. Deployment (Cloudflare Workers)
- Automatically bundle the generated TypeScript server (e.g., using esbuild)
- Deploy the bundled worker to Cloudflare Workers via their REST API
- Generate and return a unique live endpoint URL (e.g., mcp.yourapp.com/u/abc123)

5. Simple UI (Frontend)
- Hardcoded form with an input field: "Source URL" (can accept RSS or standard Website URLs)
- Submit button
- Loading state showing progress: Fetching -> Transcribing -> Chunking -> Deploying
- Result screen: Displays the deployed MCP URL with a "Copy" button
- Instructions: Clear steps on how to add the URL to Claude Desktop configuration

6. Testing & Validation
- End-to-end test with a real podcast RSS feed and a real documentation website
- Verify Claude Desktop can connect to the deployed URL and successfully answer questions based on both the podcast and website content
- Beta test with 10 users to validate the pipeline works perfectly
